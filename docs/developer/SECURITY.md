# 安全政策

PlookingII 团队非常重视项目的安全性。我们感谢安全研究人员和用户帮助我们保持项目的安全。

## 🛡️ 支持的版本

我们为以下版本提供安全更新：

| 版本 | 支持状态 | 支持截止日期 |
| ---- | -------- | ------------ |
| 1.4.x | ✅ 全面支持 | 持续支持 |
| 1.3.x | ✅ 安全更新 | 2025-12-31 |
| 1.2.x | ⚠️ 重要安全问题 | 2025-06-30 |
| < 1.2 | ❌ 不支持 | 已停止 |

### 版本说明

- **全面支持**: 提供功能更新和安全修复
- **安全更新**: 仅提供关键安全修复
- **重要安全问题**: 仅修复严重安全漏洞
- **不支持**: 不再提供任何更新

## 🚨 报告安全漏洞

### 紧急联系方式

如果您发现了安全漏洞，请**不要**通过公开的issue报告。请使用以下安全渠道：

#### 首选方式：私密报告
1. 在GitHub上使用[Private Security Advisory](https://github.com/onlyhooops/plookingII/security/advisories/new)
2. 发送邮件至：qq807948548@icloud.com
3. 使用GPG加密（公钥见下方）

#### 报告内容

请在报告中包含以下信息：

- **漏洞类型**：如代码注入、权限提升等
- **影响范围**：受影响的版本和组件
- **重现步骤**：详细的复现方法
- **概念验证**：如果可能，提供PoC代码
- **影响评估**：潜在的安全影响
- **建议修复**：如果有修复建议

### 响应时间承诺

| 严重程度 | 首次响应 | 状态更新 | 修复发布 |
|---------|---------|---------|---------|
| 严重 | 24小时 | 每48小时 | 7天内 |
| 高 | 48小时 | 每72小时 | 14天内 |
| 中 | 5天 | 每周 | 30天内 |
| 低 | 10天 | 每两周 | 60天内 |

## 🔒 安全最佳实践

### 用户安全指南

#### 安装和更新
- 始终从官方源安装PlookingII
- 定期更新到最新版本
- 验证下载文件的完整性

#### 使用建议
- 不要以管理员权限运行PlookingII
- 谨慎处理来自不可信来源的图像文件
- 定期检查和更新依赖项
- 在沙盒环境中测试新版本

#### 网络安全
- 使用安全的SMB连接（SMB3.0+）
- 避免在不安全的网络上访问远程资源
- 使用VPN连接远程存储
- 定期更改访问凭据

### 开发者安全指南

#### 代码安全
- 遵循安全编码规范
- 使用静态分析工具（bandit）
- 进行代码审查
- 验证所有输入数据

#### 依赖管理
- 定期更新依赖项
- 使用`safety`检查已知漏洞
- 锁定依赖版本
- 审查第三方库

#### 构建安全
- 使用可重现的构建
- 验证构建环境
- 签名发布文件
- 使用安全的CI/CD流水线
